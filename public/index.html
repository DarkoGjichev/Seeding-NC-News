Introduction

The NC News API is designed around REST principles, making it easy to use and understand. It features clear, resource-based URLs, accepts form-encoded request bodies, returns JSON responses, and uses standard HTTP methods and status codes.

The API provides access to data related to news articles, users, comments, and article topics. You can use it to retrieve lists of articles, fetch specific articles by ID, read and post comments, and explore topics and users.

You can also try the API in test mode, allowing you to experiment safely without affecting live data.

Articles

GET /api/articles - Returns all articles. Supports topic, sort_by, order queries.

Live endpoint: /api/articles

GET /api/articles/:article_id - Returns a single article including comment count.

Example: /api/articles/1

{
  "article_id": 1,
  "title": "Learning JavaScript",
  "topic": "coding",
  "author": "johndoe",
  "body": "JavaScript is great!",
  "created_at": "2025-08-25T12:00:00.000Z",
  "votes": 5,
  "article_img_url": "https://example.com/js.png",
  "comment_count": 2
}

Comments

GET /api/articles/:article_id/comments - Returns all comments for an article.

Example: /api/articles/1/comments

[
  {
    "comment_id": 2,
    "author": "janedoe",
    "body": "Great article!",
    "votes": 0,
    "created_at": "2025-08-25T14:00:00.000Z"
  }
]

POST /api/articles/:article_id/comments - Adds a new comment.

Request Body:

{
  "username": "janedoe",
  "body": "This is my comment"
}

Example live endpoint: /api/articles/1/comments

DELETE /api/comments/:comment_id - Deletes a comment.

Example live endpoint: /api/comments/2

Votes

PATCH /api/articles/:article_id - Update vote count for an article.

Example: /api/articles/1

Request Body:
{
  "inc_votes": 1
}

Errors

The NC News API uses standard HTTP status codes to indicate success or failure. When something goes wrong, the API returns a JSON response with a relevant status code and error message.

400 Bad Request

The request was invalid or missing required data. Usually happens when inserting the wrong data type or missing information.

Status: 400 Bad Request
Response Body:
{
  "status": 400,
  "message": "Bad Request"
}

404 Not Found

The requested resource does not exist. Often happens when searching for an article ID, comment ID, or username that doesnâ€™t exist.

Status: 404 Not Found
Response Body:
{
  "status": 404,
  "message": "Article not found"
}

500 Internal Server Error

Something went wrong on the server side.

Status: 500 Internal Server Error
Response Body:
{
  "status": 500,
  "message": "Internal Server Error: Please try again later"
}
