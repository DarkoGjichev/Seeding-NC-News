Introduction

The NC News API is designed around REST principles, making it easy to use and understand. It features clear, resource-based URLs, accepts form-encoded request bodies, returns JSON responses, and uses standard HTTP methods and status codes.

The API provides access to data related to news articles, users, comments, and article topics.

You can also try the API in test mode, allowing you to experiment safely without affecting live data.

Errors

The NC News API uses standard HTTP status codes to indicate the success or failure of a request. When something goes wrong, the API returns a clear error message in the response body, along with a relevant status code.

Common error responses include:

400 Bad Request: The request was invalid or missing required data.

Usually happens when inserting wrong data type or not including all the information for the query

404 Not Found: The requested resource doesn't exist.

Mostly happens when searching for id of article,id of comment or a username that doesn't exists

500 Internal Server Error: Something went wrong on our end.

These responses are designed to help you quickly understand what went wrong and how to fix it.

Paths

GET /api/topics

Returns 200 and all topics.

{
    "slug": "coding",
    "description": "Code is love, code is life",
    "img_url": ""
}

GET /api/articles

Returns 200 and all articles with information about the articles, datestamp and image.

{
    "article_id": 34,
    "title": "The Notorious MSG’s Unlikely Formula For Success",
    "topic": "cooking",
    "author": "grumpy19",
    "created_at": "2020-11-22T11:13:00.000Z",
    "votes": 0,
    "article_img_url": "https://article-image",
    "comment_count": 11
}

Some possible queries you can add to this path are:

  • /api/articles?sort_by=[collumn_name] - To sort the articles, by default it is sorted by the datestamp in descending order
  • /api/articles?topic=[topic] - To filter through articles by topic

GET /api/users

Returns code 200 and all users.

{
    "username": "tickle122",
    "name": "Tom Tickle",
    "avatar_url": "https://avatar-url"
}

GET /api/articles/:article_id

Returns code 200 and the article with given ID number

PATCH /api/articles/:article_id

{
    "article_id": 8,
    "title": "Express.js: A Server-Side JavaScript Framework",
    "topic": "coding",
    "author": "cooljmessy",
    "body": "You’re probably aware that JavaScript ...",
    "created_at": "2020-10-05T22:23:00.000Z",
    "votes": 0,
    "article_img_url": "https://article-img-url",
    "comment_count": 7
}

Returns 202 and returns the updated votes

GET /api/articles/:article_id/comments

Returns code 200 and all comments for selected article.

POST /api/articles/:article_id/comments

post new comment if you are signed in and returns 201 with the new comment

DELETE /api/comments/:comment_id

Deletes selected comment and returns code:204
